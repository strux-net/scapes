tex_tru - A tru text processor

../common.tru
This is one of a family of textproc-truers.
Current members are
html_tru for html
tex_tru for tex/latex/pdf
Both share the same syntax-idea.

Some style-vars can be set to modify the output.
option -show_vars shows the compiled in settings.

Your document may consist of more than one input-file.
Also the style-settings can be in a separate file.

See style for more details.
tex.tru
The created output is valid tex.
The simplest sequence of commands is:
tex_tru mydoc.txtTR >mydoc.tex
latex mydoc.tex
Another sequence may look like
tex_tru style intro chapter1 chapter2 chapter3 >mybook.tex
pdflatex mybook.tex

usage : tex_tru [Options] [<file> ....] produces a tex-file find details in ~/strux/doc/tex_tru.html or if you do not have the strux-tools installed http://www.strux.net/strux_zip/strux/doc/tex_tru.html per convention the input files have the extension .txtTR The input consists of one or more input files. freetextOrSection

freetextOrSectionmandatory.+represents the line

If this line introduces a new block, then it is considered a section.
Example : [ help ] [ cmd ]

this is new section this is a line inside the section this is a new section this is a line inside the new section
In case of a section:
tex.tru
The style-var Sections is an array containing the strings for the corresponding level.
If more levels are present than entries in this array, then the last entry is used for these.
Say the chosen entry is ‘\chapter‘, then it results in
\chapter{whatever the section is}
A label is set after this statement, so you can refer to this chapter by naming it.
The content of this chapter is additionally surrounded by the style-vars sectionStart and sectionEnd.

The complete sequence is:
1 \chapter{whatever the section is}
2 \label{internal representation of "whatever the section is"}
3 sectionStart
4 content of the section (chapter, paragraph or whatever)
5 sectionEnd

style_controlled

style_controlledmandatory.+represents the line

What such an action does is:
take the name of the structure-item, append Start and take this as the name of the style-var to be used at the start.
take the name of the structure-item, append End and take this as the name of the style-var to be used at the end.
